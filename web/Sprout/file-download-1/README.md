# 📂 Case Analysis: file-download-1

## 1. 문제 정보 (Challenge Info)
- **Description**: 파일 다운로드 및 읽기 기능에서 발생하는 Path Traversal(경로 탐색) 취약점을 이용하여 서버 내의 권한 밖 파일을 읽어내는 문제입니다.
- **Target**: 드림핵 워게임 서버 (file-download-1)
- **Flag Format**: `DH{...}`

## 2. 분석 개요 (Overview)
- **Objective**: 파일 이름을 지정하는 파라미터에 `../` (Dot-Dot-Slash)를 삽입하여 지정된 다운로드 디렉터리를 벗어난 후, 서버 어딘가에 위치한 플래그 파일을 읽어옴.
- **Key Concept**: **Path Traversal** (Directory Traversal), Inconsistent Input Validation

## 3. 분석 및 해결 단계 (Steps)

### Step 1: 취약점 분석 (Root Cause)
서버에는 파일을 업로드하는 `/upload` 엔드포인트와 파일을 읽어오는 `/read` 엔드포인트가 존재합니다.

- **취약점**: `/upload` 로직에서는 파일명에 `..`이 포함되어 있는지 검사하는 블랙리스트 필터링이 존재하여 상위 경로 접근을 막고 있습니다. 하지만 `/read` 로직에서는 `..`에 대한 필터링이 누락되어 있습니다. 이로 인해 읽기 기능에서는 경로 탐색이 가능해집니다.

### Step 2: 페이로드 구성 (Payload Construction)
플래그 파일(예: `flag.py`)을 타겟으로 삼아 상위 디렉터리로 이동하는 페이로드를 작성합니다.

**Payload (상위 경로 탐색)**:
`../flag.py`

### Step 3: 공격 실행 (Exploit)
1. 필터링이 누락된 취약한 엔드포인트인 `/read` 페이지로 이동합니다.
2. 파일을 읽어오는 파라미터(예: `?name=`) 값에 구성한 페이로드를 주입하여 요청을 전송합니다.
**요청 URL**: `/read?name=../flag.py`
3. 파일이 다운로드되거나 화면에 출력되면, 그 안에서 플래그 값을 확인합니다.

## 4. 결과 (Result)
엔드포인트 간의 일관성 없는 검증 로직을 파악하고, 필터링이 없는 `/read` 기능을 통해 시스템 내부의 플래그 파일을 성공적으로 읽어왔습니다.

- **Flag**: `DH{uploading_webshell_in_python_program_is_my_dream}`

## 5. 보안 인사이트 (Retrospective)
- **Root Cause**: 기능별(Upload vs Read)로 입력값 검증 로직이 파편화되어 적용되었으며, 파일 읽기 기능에서 경로 이동 메타문자(`../`)에 대한 필터링이 누락됨.
- **Countermeasures**:
  - **특수 문자 필터링 (White-list)**: 정규표현식을 사용하여 파일명에 영문자, 숫자, 일부 안전한 기호(예: `_`, `-`)만 허용하고, 경로 탐색에 사용되는 특수 문자(`.`, `/`, `\`) 입력 자체를 원천적으로 차단.
  - **일관된 검증 로직 적용**: 파일명을 다루는 모든 엔드포인트에 대해 공통된 보안 검증 모듈을 통과하도록 강제해야 함.
  - **경로 문자열 정제**: 사용자가 입력한 파일명에서 언어별 내장 함수(예: Python의 `os.path.basename()`)를 사용하여 경로 정보를 강제로 제거하고 순수 파일명만 추출.
  - **절대 경로 검증**: 최종적으로 오픈하려는 파일의 절대 경로를 구한 뒤, 그 경로가 반드시 허용된 특정 디렉터리(예: `uploads/`) 하위에 존재하는지 검사.
